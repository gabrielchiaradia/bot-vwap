VWAP REVERSION BOT — V3

Manual consolidado de instalación, despliegue y gestión de repositorio para arquitectura multi-instancia (BTC/ETH).

1. Gestión del Repositorio (GitHub)

Para inicializar el proyecto y subirlo a un nuevo repo:


git init
git add .
git commit -m "Initial commit: VWAP Bot Multi-instance"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/bot-vwap.git
git push -u origin main

Flujo de trabajo diario (Push/Pull):


# Para subir cambios
git add .
git commit -m "Descripción del cambio"
git push origin main

# Para actualizar el VPS con cambios locales
git pull origin main

2. Instalación y Despliegue (VPS)

Asegúrate de tener Docker y Docker Compose instalados.

1. Crear archivos de entorno (basados en las llaves de Binance):

nano .env.eth y nano .env.btc

2. Construir e iniciar los contenedores:


docker-compose up -d --build

3. Comandos de Mantenimiento

AcciónComando
Ver Estado Generaldocker ps
Ver Logs en Vivo (ETH)docker logs -f vwap-eth
Ver Logs en Vivo (BTC)docker logs -f vwap-btc
Reiniciar Botsdocker-compose restart
Detener tododocker-compose down

4. Dashboard & Logs

El dashboard corre en el puerto 8081. Los datos se actualizan cada minuto.

  • Acceso: http://IP_DE_TU_VPS:8081
  • Los archivos JSON se guardan en dashboard/data/ y están separados por BOT_ID.

5. Cortacircuitos (Risk Management)

El bot tiene reglas estrictas programadas en src/risk.py:
